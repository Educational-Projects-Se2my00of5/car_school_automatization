Функциональные требования – Итерация 4
Модуль 1: P2P (Peer‑to‑Peer) оценивание
1.1. Настройка P2P‑оценивания для задания
* 1.1.1. При создании или редактировании задания преподаватель может включить режим «P2P‑оценивание».
* 1.1.2. Включение P2P‑режима добавляет следующие параметры:
   * Способ распределения рецензентов:
      * Случайный – система сама назначает каждого студента на проверку другой работы.
      * Ручной – преподаватель вручную выбирает, кто какую работу проверяет.
      * Самостоятельный выбор студентом – студент сам выбирает, чью работу оценивать (из списка доступных).
   * Анонимность:
      * Полная (оценивающий не видит оцениваемого, оцениваемый не видит оценивающего).
      * Частичная (видна только работа, но не автор).
      * Открытая.
   * Дедлайн проверки (отдельный от дедлайна сдачи работы).
   * Критерии оценивания – используются те же критерии, что и для обычного задания (см. итерацию 3).
1.2. Процесс P2P‑оценивания
* 1.2.1. После дедлайна сдачи заданий система формирует пары / группы «проверяющий → проверяемый» согласно выбранному способу распределения.
* 1.2.2. Студент (проверяющий) видит работу (или её часть) того, кого назначили, и выставляет оценки по каждому критерию.
* 1.2.3. При анонимном режиме система не отображает ни имени оцениваемого, ни имени оценивающего.
* 1.2.4. После наступления дедлайна проверки система блокирует возможность выставления/изменения P2P‑оценок для всех участников.
* 1.2.5. Если студент не выставил оценку до дедлайна, система может засчитать 0 по соответствующим критериям [a]
1.3. Командное P2P‑оценивание
* 1.3.1. Система поддерживает P2P‑оценивание между командами 
* 1.3.2. После того как команды сдали задания, каждой команде случайным образом назначается другая команда на проверку.
* 1.3.3. Если в команде есть капитан – только капитан заполняет оценку проверяемой команды.
* 1.3.4. Если капитана нет – оценку выставляют все члены команды, после чего система вычисляет среднее арифметическое по каждому критерию и использует его как итоговую оценку для проверяемой команды.
1.4. Управление P2P‑распределением
* 1.4.1. Преподаватель может в любой момент (до дедлайна проверки) переназначить проверяющих вручную.
* 1.4.2. Преподаватель может просматривать, кто кого проверяет (даже если для студентов включена анонимность).
* 1.4.3. Преподаватель может отменить любое P2P‑назначение и заменить его своим.
